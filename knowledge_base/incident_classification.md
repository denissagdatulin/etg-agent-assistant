# Правила логирования тикетов

## Строгое правило последовательности

**Type - Subtype - Source - Cause**

### Важно
Оценивай каждый случай **индивидуально**.

Проверяй не формальное совпадение слов, а смысл выбранного логирования:

- подходит ли выбранная связка `Type - Subtype - Source - Cause` к фактической ситуации;
- отражает ли она проблему по смыслу;
- не пытается ли модель признать логирование ошибочным только потому, что нашла другой тоже возможный вариант.

Если сотрудник выбрал логирование, которое подходит ситуации по смыслу, считай его **корректным**.
